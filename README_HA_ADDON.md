# Интеграция Telemt для Home Assistant

Этот репозиторий предоставляет полную интеграцию **Telemt MTProxy** в Home Assistant, состоящую из:

1. **Home Assistant Add‑on** – запускает Telemt в Docker-контейнере внутри Home Assistant OS/Supervised.
2. **Пользовательский компонент** – предоставляет сущности (переключатель, датчики) внутри Home Assistant.

## Возможности

- **Установка в один клик** через магазин аддонов Home Assistant (после добавления репозитория).
- **Полная настройка через UI** – установка секрета, портов, уровня логирования, режима сети.
- **Мониторинг в реальном времени** – датчики подключений, времени работы, трафика.
- **Управление** – переключатель для запуска/остановки прокси, сервисы для перезапуска, перезагрузки конфигурации, генерации секретов.
- **Безопасность по умолчанию** – пользователь non‑root, базовый образ distroless, файловая система только для чтения.

## Установка

### 1. Добавьте репозиторий в Home Assistant

1. Перейдите в **Настройки → Аддоны → Магазин аддонов**.
2. Нажмите на три точки (⋮) в правом верхнем углу и выберите **Репозитории**.
3. Добавьте URL этого репозитория (GitHub или GitLab).
4. Аддон **Telemt MTProxy** должен появиться в магазине.

### 2. Установите аддон

1. Нажмите на **Telemt MTProxy**.
2. Нажмите **Установить**.
3. Во время установки настройте аддон (см. ниже).
4. Запустите аддон.

### 3. Настройте аддон

Перейдите на вкладку **Конфигурация** аддона и заполните:

- **Секрет** (обязательно) – 32‑символьная шестнадцатеричная строка (сгенерируйте командой `openssl rand -hex 16`).
- **Порт** – порт прослушивания (по умолчанию `443`).
- **Включить метрики** – предоставлять метрики Prometheus на порту `9090`.
- **Включить API** – предоставлять REST API на порту `9091`.
- **Уровень логирования** – `info`, `debug`, `warn`, `error` или `trace`.
- **Режим сети** – `host` (использовать сеть хоста) или `bridge` (управляемые Docker порты).

Сохраните и запустите аддон.

### 4. Установите пользовательский компонент

#### Вариант А: HACS (рекомендуется)
1. Убедитесь, что [HACS](https://hacs.xyz) установлен.
2. Перейдите в **HACS → Интеграции**.
3. Нажмите **+ Исследовать и добавить репозитории**.
4. Найдите `telemt` и добавьте его.
5. Перезапустите Home Assistant.

#### Вариант Б: Ручная установка
1. Скопируйте папку `custom_components/telemt` в директорию `config/custom_components/` вашего Home Assistant.
2. Перезапустите Home Assistant.

### 5. Настройте интеграцию

1. Перейдите в **Настройки → Устройства и службы**.
2. Нажмите **+ Добавить интеграцию** и найдите **Telemt**.
3. Введите данные подключения:
   - **Хост**: `localhost` (если аддон работает на том же хосте) или IP вашего экземпляра Telemt.
   - **Порт**: порт API (по умолчанию `9091`).
   - **Секрет**: тот же секрет, который вы использовали в аддоне.
4. Подтвердите. Интеграция создаст сущности.

## Сущности

После настройки появятся:

- **`switch.telemt_proxy`** – включение/выключение прокси.
- **`binary_sensor.telemt_status`** – показывает, доступен ли прокси.
- **`sensor.telemt_connections`** – количество активных подключений.
- **`sensor.telemt_uptime`** – время работы прокси в секундах.
- **`sensor.telemt_traffic_in`** – входящий трафик (МБ).
- **`sensor.telemt_traffic_out`** – исходящий трафик (МБ).

## Сервисы

Интеграция предоставляет следующие сервисы:

- **`telemt.restart`** – перезапустить процесс Telemt.
- **`telemt.reload_config`** – перезагрузить конфигурацию с диска.
- **`telemt.generate_secret`** – сгенерировать новый секрет (опциональный параметр длины).

## Устранение неполадок

### Аддон не запускается
- Проверьте вкладку **Логи** аддона на наличие ошибок.
- Убедитесь, что секрет состоит ровно из 32 шестнадцатеричных символов.
- Если используется порт `< 1024`, аддону может потребоваться capability `NET_BIND_SERVICE` (уже включён).

### Интеграция не может подключиться
- Убедитесь, что API включён в конфигурации аддона.
- Проверьте, что хост и порт указаны верно (по умолчанию `localhost:9091`).
- Убедитесь, что секрет совпадает.

### Сущности не появляются
- Перезапустите Home Assistant после установки пользовательского компонента.
- Проверьте логи Home Assistant на наличие ошибок.

│   ├── config.yaml
│   ├── build.json
│   ├── rootfs/run.sh
│   ├── icon.png
│   ├── logo.png
│   └── DOCS.md
├── custom_components/telemt/ # Пользовательский компонент
│   ├── __init__.py

│   ├── const.py

│   ├── sensor.py

### Сборка аддона локально
```bash
cd addon/telemt
docker build -t telemt-ha-addon .
```

### Тестирование интеграции
Поместите папку `custom_components/telemt` в директорию `config` Home Assistant и перезапустите.
